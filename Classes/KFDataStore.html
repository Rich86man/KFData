Overview

KFDataStore is a wrapper around an NSPersistentStoreCoordinator. You would normally create a single instance of KFDataStore to use across the whole of your application. Then you would run managedObjectContextWithConcurrencyType: to get hold of a managed object context. Alternatively you can use performWriteBlock: and performReadBlock: to run a block.

Initialization

  • – init

    After using init, you will need to manually add the persistent stores. Additionally you can use the standardLocalDataStore helper method. The only reasons where you would need to manually init are if you want to support non-standard object models or custom configurations of your model.

Performing Block Operations

  • – performReadBlock:

    Asyncronously execute a read-only block on a new private context
  • – performWriteBlock:

    Asyncronously execute a block and then save the changes into the main context (and persistent store).
  • – performWriteBlock:completionHandler:

    Asyncronously execute a block and then save the changes into the main context and persistent store. But execute a completion block when this has been saved.
